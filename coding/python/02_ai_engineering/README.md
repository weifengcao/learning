# 02_ai_engineering

Purpose:
Use Python to build practical AI components with clean interfaces and production-minded habits.

Cover:

- model API clients
- structured output parsing
- schema validation
- prompt input assembly
- retries, backoff, and timeouts
- async calls and batching
- embeddings and retrieval utilities
- evaluation scripts
- logging and error reporting

Suggested exercises:

- wrap a model API behind a typed client
- validate structured JSON output with tests
- build an async batch inference utility
- score prompt outputs with a simple evaluation harness
- implement a document chunking pipeline

What this track improves:

- practical Python service code
- interface design around unreliable model behavior
- testing discipline for AI features
- operational thinking around latency, cost, and failure handling
