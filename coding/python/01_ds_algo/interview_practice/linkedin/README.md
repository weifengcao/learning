# linkedin

Purpose:
Practice representative Python interview problems for product and infrastructure companies like LinkedIn.

Important note:

This is a pattern-focused practice set, not a claim about exact current LinkedIn interview questions.

## Common emphasis areas

- graph traversal and dependency relationships
- intervals and scheduling
- top-k and ranking problems
- streaming or mutable data structure design
- string and collection normalization
- tree traversal with grouping or ordering

## Representative practice set

Graph and dependency style:

- Course Schedule
- Number of Connected Components in an Undirected Graph
- Accounts Merge

Intervals and scheduling:

- Meeting Rooms
- Meeting Rooms II
- Merge Intervals

Ranking, heaps, and feeds:

- Top K Frequent Elements
- K Closest Points to Origin
- Merge K Sorted Lists

Streaming or mutable design:

- Design Hit Counter
- Moving Average from Data Stream
- Insert Delete GetRandom O(1)

Tree and nested structure traversal:

- Binary Tree Vertical Order Traversal
- Nested List Weight Sum
- Find Leaves of Binary Tree

## Why this set is useful

This practice set is biased toward the kinds of patterns that show up in data-heavy, product-heavy, and backend-heavy environments:

- traversing structured relationships
- aggregating ordered results
- handling mutable state
- reasoning about throughput-oriented data problems

## Practice order

1. intervals and heaps
2. graph traversal
3. mutable data structure design
4. tree grouping and traversal
