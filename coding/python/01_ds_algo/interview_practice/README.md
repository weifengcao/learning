# interview_practice

Purpose:
Build Python interview readiness without turning practice into random problem collection.

Structure:

- `generic/`: broad coding interview practice
- `linkedin/`: representative product and backend-heavy practice for companies like LinkedIn
- `PRACTICE_TEMPLATE.md`: a standard way to solve and review each problem

Important note:

The company-style folders are not meant to claim exact current interview questions. They are pattern-weighted practice sets built around the kinds of problems commonly associated with product and infrastructure engineering interviews.

## Recommended workflow

For each problem:

1. Restate the problem clearly.
2. List constraints and edge cases.
3. Write a brute-force approach first if helpful.
4. Improve to the target solution.
5. State time and space complexity.
6. Write a clean Python implementation.
7. Add a few test cases, including edge cases.
8. Note one follow-up or variation.

## Output standard

Each solved problem should include:

- problem name
- pattern
- final complexity
- clean Python solution
- test cases
- short note on what was easy to miss

## Practice target

A good starting target is:

- `generic/`: 12-15 problems
- `linkedin/`: 8-10 problems

Keep the solution bar high. Fewer clean solutions are more valuable than many rushed ones.
