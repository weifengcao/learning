# 03_projects

Purpose:
Combine strong Python implementation with realistic end-to-end exercises.

Project criteria:

- must include non-trivial Python logic
- must have tests for core behavior
- must have a short README describing tradeoffs
- should combine algorithmic rigor and practical workflow design

Suggested projects:

- retrieval pipeline with chunking, indexing, and scoring
- tool-calling agent runner with validation and retries
- dataset deduplication and ranking utility
- evaluation harness for prompt or workflow regression checks
- support-ticket classification service with structured output parsing

Review checklist:

- is the core logic correct
- are interfaces clear
- are edge cases tested
- is error handling explicit
- can the code be extended without major rewrites
