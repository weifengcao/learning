# 03_workflows_and_agents

Purpose:
Design controlled workflows first, then introduce agents only where they are justified.

Cover:

- deterministic workflow orchestration
- approval gates and human-in-the-loop design
- tool use boundaries
- retries, compensating actions, and rollback
- memory scope and state management
- planner-executor patterns
- single-agent vs multi-agent tradeoffs
- action safety and blast-radius reduction
